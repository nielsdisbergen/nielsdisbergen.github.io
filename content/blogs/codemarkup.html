Using Source Code Syntax Highlighting

Posted 10 December 2016


Syntax highlighting is one of the basics to be implemented on web pages featuring some form of source code displaying. There are many different ways to implement such color-coding and formatting, I will have a brief look at several options though mainly focus on Google's Code Prettifier.


Webapp

Converting (short) code snippets into pretty-printed copy-paste HTML can be achieved very easilly using the hilite.me webapp. Simply copy your source code in the input box, set the language, adjust style and line-number inclusion if you desire, and hit the "Highlight!" button to generate that pretty code and get and instant preview. 'Simply' copy-paste HTML or CSS into your page and you are ready to go.


JavaScript based

Several packages are available to perform code highligting, among which highlight.js, which turned 10 in August 2016. Many different languages are supported and styles available, combined with automatic language detection and multi-language highligting capabilities. Even though highlight.js is a very nice package, my personal preference goes to Google's Code Prettifier. Overall it is very fast, well implemented, supports many languages and styles, is easilly served, and styles can be created and managed nicely via CSS.

Prettify auto-loader

Prettify's CSS and JavaScript can be quickly loaded on the fly by placing the auto-loader in your script section, just before the body end 

    <script src="https://cdn.rawgit.com/google/code-prettify/master/loader/run_prettify.js"></script>
  </body>
It is possible to provide options as arguments, for example styling code with their Desert skin and forcing html 
  <script src="https://cdn.rawgit.com/google/code-prettify/master/loader/run_prettify.js?lang=html&amp;skin=desert"></script>

After adding the auto-loader, using Prettify becomes as simple as classing a <pre>, <code>, or <xmp> with prettyprint and the prettyprinter will do all the magic 

  <pre class="prettyprint">
        <!--body goes here-->
  </pre>

Prettify serving

Serving prettify requires a little more effort and code, on the upside it is (most of the times) faster and allows tweaking of styles and code where desired. Aside from source files, prettify only needs the JavaScript and CSS which is employed in the pretty print class text, hence serving of a sub-set is possible. If, for example, you want to serve LaTeX only code with standard styling, the minimum Prettify set would be run_prettify.js, prettify.js, lang-latex.js, and prettify.css

In order to get a hosted version running, a few further declarations will have to be made. First, linking the JavaScript prettify.js file in the script section just before the body end: 

  <script type="text/javascript" src="prettify.js"></script>
</body>

Next, link the desired CSS file to your HTML in the header section. Linking prettify.css will result in default color formatting; in case you would like to make use of any other theme, simply link your CSS of choice instead. 

  <link href="prettify.min.css" type="text/css" rel="stylesheet"/>
Some great Prettify color themes can be found on Yoshihide Jimbo's blog, this page, for example, has been formatted with the tomorrow-night-eighties theme found there.

When your webpage finished loading, the prettyPrint function needs to be ran; one possibility is to declare this via the onload handler in the body header: 

  <body onload="prettyPrint()">

More detailed information can be found on the Prettify Github, or for a more general guide have a look at this blog post.
Happy pretty-printing!
