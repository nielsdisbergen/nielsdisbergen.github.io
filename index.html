Who

A Neuroscientist by training with a great interest in and fascination for software development and technology. Currently a doctoral (PhD) candidate at the Brain Imaging Center of Maastricht University (MBIC), working on sound processing in the human brain. More specifically, investigating how the brain separates sounds in an environment where multiple sounds are presented simultaneously. For my research I make use of ultra-high field functional Magnetic Resonance Imaging at 7 Tesla combined with advanced data analysis techniques such as machine learning. For a more detailed description see About Me.

Why Brains

Brains are fascinating in many ways, they are so much more than merely fantastically efficient processing units. Understanding how brains work will most probably increase our understanding of many processes on the way. One of the main obstacles to understanding brains is not so much a technical one, for the time being it is more philosophical: can the being itself understand how its self operates? Maybe the only way to understand the mechanisms would be by virtue of developing 'artificial intelligence' capable of surpassing our levels of understanding and able to express their 'knowledge' in dimensions we can grasp...

Developing Towards Neuro-Inspired Algorithms

Software is an inherent part of most people's life, which is no different in the greater part of modern scientific disciplines. Without software combined with scientific minds and computational power, virtually no scientific applications could have been developed. Employing any data analysis method does not merely require computational strength, it strongly calls for appropriately built software. Developing scientific software is not about producing large chunks of code which merely work and others can somehow employ, it needs reliable, stable, and fast code which is not only properly developed, but also rigorously checks data and properly handles exceptions where applicable. The further scientific analysis methods progress, the less they can be grasped by those working at more applied, less technical levels, the more need for good software increases. One of my professional interests lies exactly there, in building software which helps others excel in their scientific/professional environment by being able to focus on tasks other than (re-)writing code themselves for processes which can very well be automated.
